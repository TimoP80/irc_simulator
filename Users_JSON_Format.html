üèóÔ∏è JSON Structure

The JSON format consists of an array of user objects, where each user object contains all necessary information for AI simulation.

{ "users": [ { "nickname": "string", "status": "online" | "away", "personality": "string", "languageSkills": { ... }, "writingStyle": { ... } } ] }

Note: The root level is an array of user objects, not wrapped in a "users" property. Each user object represents one virtual user in the simulation.

Property Type Required Description
nickname string ‚úÖ Yes Unique identifier for the user (lowercase, no spaces)
status string ‚úÖ Yes Current status: "online" or "away"
personality string ‚úÖ Yes Detailed personality description for AI behavior
languageSkills object ‚úÖ Yes Language proficiency and preferences
writingStyle object ‚úÖ Yes Communication style preferences

‚úÖ Validation Rules

When importing user data, the following validation rules are applied:

Required Fields

  • nickname must be a non-empty string
  • status must be either "online" or "away"
  • personality must be a non-empty string
  • languageSkills object must be present
  • writingStyle object must be present

Language Skills Validation

  • fluency must be one of: "beginner", "intermediate", "advanced", "native"
  • languages must be an array with at least one language
  • accent is optional but must be a string if provided

Writing Style Validation

  • formality must be one of: "casual", "formal", "mixed"
  • verbosity must be one of: "concise", "moderate", "verbose"
  • humor must be one of: "none", "light", "heavy"
  • emojiUsage must be one of: "none", "minimal", "frequent"
  • punctuation must be one of: "minimal", "standard", "excessive"

‚ö†Ô∏è Important Notes

‚Ä¢ Invalid or missing required fields will cause the user to be skipped during import
‚Ä¢ The status field is automatically set to "online" for imported users
‚Ä¢ Duplicate nicknames are not allowed - the last occurrence will be used
‚Ä¢ All string values are trimmed of whitespace
