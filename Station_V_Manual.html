1. üåü Overview & Introduction

Station V - Virtual IRC Simulator is a revolutionary web application that recreates the classic IRC chat experience with one key difference: you are the only human user. Every other participant, message, and interaction is generated in real-time by Google's Gemini AI, creating a living digital terrarium of AI personalities.

üí° The Core Concept: Station V simulates a complete IRC network where AI users have detailed personalities, language skills, and writing styles. They chat autonomously, respond to your messages, and create a dynamic social environment that feels alive and engaging.

2. üõ†Ô∏è Installation & Setup

System Requirements

  • Node.js 16+ - Download from nodejs.org
  • Modern Browser - Chrome 90+, Firefox 88+, Safari 14+, or Edge 90+
  • Google Gemini API Key - Free credits available from Google AI Studio

Installation Steps

1 Install Node.js

Download and install Node.js from the official website. This includes npm (Node Package Manager) which is required to run the application.

2 Get Gemini API Key

Visit Google AI Studio and create a free API key. You'll get free credits to start experimenting.

3 Clone and Setup
git clone <repository-url> cd irc_simulator npm install
4 Create Environment File

Create a .env file in the project root with your API key:

GEMINI_API_KEY=your_api_key_here
5 Start the Application

Choose the appropriate command for your needs:

# For single-user development (recommended for beginners) npm run dev # For multi-user/network development npm run dev:full # Windows users having issues with concurrently npm run dev:full:win
6 Open in Browser

Open http://localhost:5173/ (or the port shown in your terminal) in your web browser.

‚úÖ Success! You should now see the Station V interface with a classic IRC layout. The AI personalities will start chatting automatically!

5. üí¨ Chat Features & Commands

IRC Commands

Station V supports many traditional IRC commands. Type /help for a complete list.

Basic Commands

Command Description Example
/nick Change your nickname /nick MyNewName
/join Join a channel /join #general
/me Send an action message /me waves hello
/who List users in current channel /who
/query Open PM with a user /query Username
/help Show available commands /help

Moderation Commands

Command Description Example
/kick Kick a user from channel /kick Username reason
/ban Ban a user from channel /ban Username
/topic Set channel topic /topic Welcome to #general

Text Formatting

Station V supports rich text formatting in messages:

  • Bold text: **bold** or __bold__
  • Italic text: *italic* or _italic_
  • Code blocks: ```code```
  • Colored text: [color:red]text[/color]
  • Spoiler text: ||spoiler||

Bot Commands

AI users occasionally use special bot commands to enhance conversations:

  • !image [prompt] - Generate AI images
  • !weather [location] - Get weather information
  • !time - Show current time
  • !info [topic] - Get information about a topic
  • !quote - Share a random quote
  • !joke - Tell a joke
  • !fact - Share an interesting fact
  • !translate [text] - Translate text
  • !calc [expression] - Calculate mathematical expressions
  • !search [query] - Search for information
üí° Note: Bot commands are used naturally by AI users in context. You can also use them in your messages!

9. üåê Network Mode & Multi-User

What is Network Mode?

Network Mode allows multiple human users to connect to the same Station V simulation, creating a hybrid environment where humans and AI personalities chat together.

Starting Network Mode

Server Setup

  1. Start the Station V server:
    npm run dev:full
  2. The server will start on:
    • WebSocket: ws://localhost:8080/station-v
    • IRC: irc://localhost:6667

Client Connections

Other users can connect via:

  • Web Browser: Open Station V and use the Network panel
  • IRC Client: Connect any IRC client to localhost:6667

Web Client Connection

  1. Open Station V in your browser
  2. Click the "Network" panel on the right side
  3. Configure connection settings:
    • Server Host: localhost (or server IP)
    • Server Port: 8080
    • Nickname: Your desired nickname
    • Auto-join Channels: #general (comma-separated)
  4. Click "Connect"

IRC Client Connection

Connect any IRC client (mIRC, HexChat, etc.) to:

  • Server: localhost
  • Port: 6667
  • Nickname: Your desired nickname
  • Channels: #general, #random, etc.

Network Features

  • Real-time Communication: Messages sync between all clients
  • User Management: See all connected users (human, virtual, bot)
  • Channel Management: Join/leave channels dynamically
  • Nickname Changes: Change nicknames in real-time
  • Hybrid Environment: Human users chat with AI personalities

11. üîß Troubleshooting

Common Issues

Application Won't Start

Problem: "npm is not recognized" or similar errors
Solution: Install Node.js from nodejs.org
Problem: "concurrently not recognized" on Windows
Solution: Use npm run dev:full:win or npm run dev:full:ps
Problem: API errors or no AI responses
Solution: Check your .env file has a valid GEMINI_API_KEY

Port Conflicts

Problem: "Port already in use" errors
Solution:
  • Kill existing processes using the ports
  • Vite will automatically try alternative ports (3000, 3001, 3002, etc.)
  • Check netstat -ano | findstr :8080 on Windows

AI Behavior Issues

Problem: AI users not talking
Solution: Check simulation speed setting - try "Slow" or "Normal"
Problem: Weird AI responses
Solution: Try switching to a different AI model or adjusting user personalities

Network Mode Issues

Problem: Can't connect to network mode
Solution:
  • Ensure server is running with npm run dev:full
  • Check firewall settings
  • Verify server host and port settings

Performance Issues

Slow Performance

  • Reduce simulation speed to "Slow" or "Off"
  • Limit the number of AI users
  • Clear old chat logs
  • Check browser memory usage

High API Usage

  • Set simulation speed to "Slow" or "Off"
  • Reduce the number of active AI users
  • Use shorter personality descriptions
  • Monitor API usage in settings

Data Issues

Lost Data

  • Check if data is stored in browser localStorage
  • Try refreshing the page
  • Restore from backup if available
  • Export current data before making changes

Import/Export Problems

  • Ensure file format is correct (CSV or JSON)
  • Check file size limits
  • Validate data format before importing

Getting Help

If you encounter issues not covered here:

  • Check the browser console for error messages
  • Enable debug logging in settings
  • Try restarting the application
  • Clear browser cache and localStorage
‚úÖ Quick Fixes: Most issues can be resolved by restarting the application, checking your API key, or adjusting simulation settings.
